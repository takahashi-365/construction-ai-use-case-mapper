# 協議用参考資料方針

このドキュメントでは、PoC 2: BIM / Construction AI Use Case Mapper における `discussion_reference_report_v001.md` の位置づけと作成方針を定義します。

本PoCでは、AI/DX活用分類結果を最終判断として扱いません。
分類結果、DXサービス候補、深掘り対象、確認事項は、関係者との協議に使うための参考資料として扱います。

---

## 1. 目的

協議用参考資料の目的は、建設・BIM関連業務のユースケースについて、AI/DX活用の可能性を関係者と検討しやすくすることです。

具体的には、以下を整理します。

```text
どの業務がAI/DX活用候補になるか
どのAI/DX支援パターンが考えられるか
どの業務に人間レビューが必要か
どの業務を追加ヒアリングすべきか
どの関係者に確認すべきか
どのような論点を協議すべきか
```

この資料は、AI/DXサービスの最終提案書ではありません。
あくまで、初期整理後の協議を進めるための中間成果物です。

---

## 2. 基本方針

本PoCでは、協議用参考資料を以下の方針で作成します。

```text
AI分類結果を最終判断として扱わない
AI/DX活用候補を協議材料として整理する
人間レビューが必要な理由を明記する
深掘りが必要な理由を明記する
関係者に確認すべき質問を整理する
AIに任せてよい範囲と、人間が判断すべき範囲を分ける
実案件データ、顧客名、社内サービス詳細は使用しない
```

---

## 3. 協議用参考資料の位置づけ

`discussion_reference_report_v001.md` は、以下のような位置づけです。

```text
入力CSV：
建設・BIM関連業務ユースケースの初期整理

分類処理：
RAG、BI、ルールベースチェック、自動化支援、人間レビュー、深掘り対象に分類

協議用参考資料：
分類結果をもとに、関係者と次に何を確認・検討するかを整理する資料
```

つまり、協議用参考資料は、以下の橋渡しを行います。

```text
広く浅い初期整理
→ AI/DX活用候補の分類
→ 関係者との協議
→ 狭く深い追加ヒアリング
→ DXサービス候補の具体化
```

---

## 4. レポートに含める内容

`discussion_reference_report_v001.md` には、以下の内容を含めます。

```text
レポートの目的
AI分類結果の扱い方
全体サマリー
DXサービス候補一覧
RAG候補ユースケース
ルールベースチェック候補ユースケース
自動化支援候補ユースケース
人間レビュー必須ユースケース
深掘り対象ユースケース
関係者に確認すべき質問
注意事項
```

---

## 5. AI分類結果の扱い方

レポート内では、AI分類結果について以下を明記します。

```text
分類結果は最終判断ではない
分類結果は自動承認ではない
分類結果は専門家判断の代替ではない
分類結果は関係者協議のための参考資料である
```

特に、以下に関わる業務では、人間レビューを必須とします。

```text
設計判断
施工判断
法規判断
安全判断
契約判断
コスト影響判断
責任範囲の判断
外部提出物の最終承認
```

---

## 6. DXサービス候補の扱い方

DXサービス候補は、AI/DX活用の方向性を示すための初期候補です。

例：

```text
Rule-based quality checker
RAG assistant
BI dashboard
Automation support tool
Human-reviewed AI drafting support
Human-reviewed risk review support
```

ただし、これらは最終決定ではありません。

DXサービス候補は、以下を確認したうえで検討します。

```text
業務目的
関係者
入力データ
出力データ
判断基準
責任範囲
例外処理
人間レビューの必要性
実装可能性
```

---

## 7. 人間レビュー必須ユースケースの扱い方

`HumanReviewRequired=True` のユースケースは、AI/DX活用ができない業務という意味ではありません。

むしろ、以下のような人間レビュー前提のAI支援として扱います。

```text
情報整理
要約
分類
候補提示
ドラフト作成
確認項目の抽出
不整合候補の抽出
```

レポートでは、各ユースケースについて以下を示します。

```text
なぜ人間レビューが必要か
どの関係者が確認すべきか
AIに任せてよい範囲はどこか
人間が最終判断すべき範囲はどこか
```

---

## 8. 深掘り対象ユースケースの扱い方

`DeepDiveRequired=True` のユースケースは、次に追加ヒアリングや詳細確認を行うべき対象です。

深掘り対象になる理由は、主に以下です。

```text
判断基準が明文化されていない
正とする資料が決まっていない
関係者が複数いる
責任範囲が不明確である
入力データ形式が案件ごとに異なる
例外処理が多い
外部提出物や契約責任に関係する
設計判断、施工判断、法規判断を含む
```

レポートでは、各ユースケースについて以下を示します。

```text
深掘りが必要な理由
確認すべき関係者
確認すべき質問
追加で必要な情報
```

---

## 9. 協議時に確認すべき観点

協議では、以下の観点を確認します。

### 9.1 業務目的

```text
この業務の目的は何か
成果物は誰が使うのか
成果物は何の判断に使われるのか
業務の失敗は何に影響するのか
```

---

### 9.2 入力データ

```text
使用する入力データは何か
データ形式は安定しているか
案件ごとに形式が変わるか
正とする資料はどれか
データは誰が作成・更新するか
```

---

### 9.3 出力データ

```text
成果物は何か
誰が成果物を確認するか
成果物は外部提出されるか
成果物は承認や契約に関係するか
どの形式で出力するのがよいか
```

---

### 9.4 判断基準

```text
判断基準は明文化されているか
ルールは案件ごとに変わるか
例外処理はあるか
最終判断者は誰か
AIが候補を出してよい範囲はどこまでか
人間が必ず確認すべき範囲はどこか
```

---

### 9.5 関係者

```text
誰に確認すべきか
誰が最終判断するか
誰がデータを作成しているか
誰が成果物を利用するか
誰が承認するか
誰が責任を持つか
```

---

### 9.6 AI/DX化の方向性

```text
RAGで参照すべき文書はあるか
BIで可視化すべき指標はあるか
ルールベースチェックにできる基準はあるか
自動化できる繰り返し作業はあるか
人間レビューをどこに入れるべきか
```

---

## 10. レポートで避ける表現

協議用参考資料では、以下のような断定表現を避けます。

```text
AIで判断できる
AIで自動承認できる
AIが正しい答えを出す
このDXサービスを導入すべき
この業務は自動化すべき
人間確認は不要である
```

代わりに、以下のような表現を使います。

```text
AI/DX活用候補として整理する
関係者との協議が必要である
人間レビューを前提とする
追加ヒアリングが必要である
候補として検討する
判断材料として扱う
```

## 13. 本PoCにおける重要な前提

本PoCでは、以下を前提とします。

```text
AI分類結果は最終判断ではない
DXサービス候補は初期候補である
協議用参考資料は提案書ではない
深掘り対象は、次に確認すべき業務を示すためのもの
人間レビューは、安全なAI/DX活用の前提である
実案件データ、顧客名、社内サービス詳細は使用しない
```

---

## 14. まとめ

協議用参考資料は、AI/DX活用候補を整理するための中間成果物です。

この資料の目的は、AIが答えを出すことではなく、関係者が次に何を確認し、どの業務を深掘りし、どのAI/DX支援パターンを検討するかを整理することです。

PoC 2では、協議用参考資料を通じて、以下の流れを表現します。

```text
広く浅く業務ユースケースを整理する
AI/DX活用パターンを分類する
DXサービス候補を整理する
人間レビューが必要な業務を明確にする
深掘り対象を抽出する
関係者との協議に進む
狭く深くAI/DXサービス化の可能性を検討する
```

この流れにより、建設・BIM業務を、現実的かつ安全にAI/DX活用へ接続するための前段階を表現します。
